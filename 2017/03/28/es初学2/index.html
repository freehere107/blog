发表于 2017-03-28 |

Elasticsearch

Elasticsearch是一个基于Apache Lucene(TM)的开源搜索引擎。无论在开源还是专有领域，Lucene可以被认为是迄今为止最先进、性能最好的、功能最全的搜索引擎库。

Loading the Sample Dataset 加载样本数据集

# 使用示例上的https://raw.githubusercontent.com/elastic/elasticsearch/master/docs/src/test/resources/accounts.json#数据
curl -XPOST 'localhost:9200/bank/account/_bulk?pretty&refresh' --data-binary "@accounts.json"
curl 'localhost:9200/_cat/indices?v'
返回
health status index    uuid                   pri rep docs.count docs.deleted store.size pri.store.size
yellow open   bank     X5dbpKDcS8i1KrV22Basug   5   1       1000            0    640.3kb        640.3kb

The Search API 查询api

# 这里使用Restful Api进行查询
# 用于搜索的REST API可以以_search作为关键字进行查询，示例返回银行索引中的所有文档
curl -XGET 'localhost:9200/bank/_search?q=*&sort=account_number:asc&pretty&pretty'
# 重要的是要明白，获得了搜索结果，Elasticsearch就完成了这次请求，并且不会保留任何类型的服务器端资源
# 这与许多其他平台（如SQL）形成不同，您可能最初会先查询到查询结果的部分子集，然后要获取（或浏览）其余部分

Introducing the Query Language

# Elasticsearch提供了一种可用于执行查询的JSON风格的语言,称为查询DSL。
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "query": { "match_all": {} }
}
'
#  match_all查询只是搜索指定索引中的所有文档
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "query": { "match_all": {} },
  "size": 1
}
'
# size配置查询的记录数量，如果不配置，默认为10
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "query": { "match_all": {} },
  "from": 10,
  "size": 10
}
'
# from配置了查询的偏移量，这将查询序列为11-20的数据，默认为0
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "query": { "match_all": {} },
  "sort": { "balance": { "order": "desc" } }
}
'
# sort配置了查询的顺序，类似SQL,以balance字段倒叙排列。

Executing Searches 执行搜索

# 如果只想让个别字段返回，而不是所有字段，如示例:
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "query": { "match_all": {} },
  "_source": ["account_number", "balance"]
}
'
# 则只会返回类似这样的结果,查询语句类似SQL的select from 
{
        "_index" : "bank",
        "_type" : "account",
        "_id" : "222",
        "_score" : 1.0,
        "_source" : {
          "account_number" : 222,
          "balance" : 14764
        }
 }
 #下面基本的字段搜索查询（即针对特定字段或一组字段进行搜索),下面查询了account_number为20的记录
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": { "match": { "account_number": 20 } }
 }
 '
 # 下面查询了address为包含了mill或lane的记录
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": { "match": { "address": "mill lane" } }
 }
 '
 # bool查询允许使用布尔逻辑将更小的查询组合成较大的查询。
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": {
     "bool": {
       "must": [
         { "match": { "address": "mill" } },
         { "match": { "address": "lane" } }
       ]
     }
   }
 }
 '
 # 在上面的例子中，bool must子句指定文档被认为是一个匹配的所有查询必须是真的,类似SQL where and.即address需包含mill和lane
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": {
     "bool": {
       "should": [
         { "match": { "address": "mill" } },
         { "match": { "address": "lane" } }
       ]
     }
   }
 }
 '
 # bool should子句指定一个查询列表，对于一个被认为是一个匹配的文档，它们必须是true。类似SQL where or.即address需包含mill或lane
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": {
     "bool": {
       "must_not": [
         { "match": { "address": "mill" } },
         { "match": { "address": "lane" } }
       ]
     }
   }
 }
 '
 # must_not 即must 的反义，即SQL where not ，即address不包括mill和lane的任意
 # 可以在bool查询中同时结合使用must，should和must_not子句
 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": {
     "bool": {
       "must": [
         { "match": { "age": "40" } }
       ],
       "must_not": [
         { "match": { "state": "ID" } }
       ]
     }
   }
 }
 '
 # Executing Filters 过滤器

 curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
 {
   "query": {
     "bool": {
       "must": { "match_all": {} },
       "filter": {
         "range": {
           "balance": {
             "gte": 20000,
             "lte": 30000
           }
         }
       }
     }
   }
 }
 '
# 上面示例,余额大于或等于20000且小于或等于30000的账户,即20000<=balance<=30000。

聚合查询

# 聚合的最简单的方法是将其大致等同于SQL GROUP BY和SQL聚合函数
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "size": 0,
  "aggs": {
    "group_by_state": {
      "terms": {
        "field": "state.keyword"
      }
    }
  }
}
'
# 上面会返回类似SQL：SELECT state, COUNT(*) FROM bank GROUP BY state ORDER BY COUNT(*) DESC
# 即按计数降序排序只返回state字段的前10个数据
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "size": 0,
  "aggs": {
    "group_by_state": {
      "terms": {
        "field": "state.keyword",
        "order": {
          "average_balance": "desc"
        }
      },
      "aggs": {
        "average_balance": {
          "avg": {
            "field": "balance"
          }
        }
      }
    }
  }
}
'
# 上面会以按照balance字段avg降序对平均余额进行排序
curl -XGET 'localhost:9200/bank/_search?pretty' -H 'Content-Type: application/json' -d'
{
  "size": 0,
  "aggs": {
    "group_by_age": {
      "range": {
        "field": "age",
        "ranges": [
          {
            "from": 20,
            "to": 30
          },
          {
            "from": 30,
            "to": 40
          },
          {
            "from": 40,
            "to": 50
          }
        ]
      },
      "aggs": {
        "group_by_gender": {
          "terms": {
            "field": "gender.keyword"
          },
          "aggs": {
            "average_balance": {
              "avg": {
                "field": "balance"
              }
            }
          }
        }
      }
    }
  }
}
'
#这个例子演示了按年龄段（20-29岁，30-39岁和40-49岁），然后按性别分组，然后最终得到每个年龄段的平均账户余额
